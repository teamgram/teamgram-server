# 安全策略

本文档说明漏洞报告流程、支持版本及披露原则。根目录 [SECURITY.md](../SECURITY.md) 提供简要入口。

## 漏洞报告

- **请勿在公开 issue 中披露未修复的安全漏洞**。
- 请通过 Telegram 联系项目 [作者](https://t.me/benqi)（见 README Notes），或使用 SECURITY.md 中后续提供的联系方式私下报告。

## 支持范围

- 安全更新针对**当前维护中的主版本**（以 release/tag 与 CHANGELOG 为准）。
- 已停止维护的旧版本不提供安全补丁；建议升级到受支持版本。

## 披露原则

- 在漏洞**修复并发布**后再在 CHANGELOG 或公开 issue 中说明（不提前披露细节）。
- 报告者若希望被致谢，可在报告中说明，由维护者在发布说明中注明。
